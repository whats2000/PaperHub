# Plan F6.2 — Manual slide-LaTeX editing + per-page "Sources" strip (Implementation Plan)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Two SlidesPanel capabilities that share the deck-edit + grounding plumbing:

1. **Manual LaTeX editing** — the user hand-edits the deck source and recompiles, for tweaks the agent shouldn't (or can't) make. Two entry points:
   - **Edit current frame** — a pencil toggle at the panel's top-right corner (mirroring the speaker-note pencil) swaps the slide-image area for a CodeMirror editor of *the current page's frame*; Save splices that frame back into `deck.tex` and recompiles the whole deck.
   - **Edit all deck** — a header button left of the Present button opens the same editor on the *entire* `deck.tex` (preamble + title + every frame); Save recompiles it as-is.
   The editor is **not live** — Save enters the loading mask, recompiles in the background, reloads the deck, and **stays on the current active page**. Edits are applied **verbatim** (mechanical recompile — no LLM cleanup). A `pdflatex` failure keeps the last-good deck and surfaces the error log in the editor.

2. **Per-page "Sources" strip** (the deferred F6.2 Sources panel) — the currently-empty band between the slide image and the speaker-note pane shows, for the on-screen page, a compact row of clickable chips resolved from the already-stored `deck_slides.source_sections_json` (`[{paper_id, section_name, chunk_ids}]`). Click → Citation Canvas at that chunk. An *unsourced* cite (empty `chunk_ids`) renders muted + non-clickable (the visible "cited a section with no evidence" signal); a structural/title page shows a quiet empty state.

**Already shipped (do NOT rebuild):** the per-slide grounding **storage + resolution** — `deck_slides.source_sections_json`, `sl_cite.with_grounding`, and the GENERATE/EDIT/RESTORE write paths that populate it — landed earlier (the revert-and-re-add of grounding onto the one-shot pipeline). This plan delivers the remaining F6.2 **read endpoint + UI**, plus the new manual-editing capability. (Note: grounding shipped as the richer `source_sections_json` = `[{paper_id, section_name, chunk_ids}]`, NOT the originally-specced flat `grounding_chunk_ids_json` int array — the SRS §III-5.3 block is reconciled in the same branch.)

**Tech Stack:** Python 3.11 · FastAPI · `aiosqlite` · pytest/pytest-asyncio · `uv` (backend) · React 19 · TypeScript strict · Zustand · `@uiw/react-codemirror` + `@codemirror/lang-*` · Vitest + RTL + MSW · `react-i18next` (frontend).

**Authoritative spec:** SRS §III-5.3 "Per-slide grounding traceback" + the new "Manual slide editing" block (reconciled in this branch).

**Conventions:** backend cmds from `backend/` via `uv` (never pip/system python); frontend from `frontend/` via `npm`; PowerShell `;` chaining; strict ruff + mypy + tsc + ESLint; **TDD (failing test first) per task, commit per task, no push**; every new UI string via `t("<ns>:<key>", "English fallback")` in **all four** locales (`en` source + `zh-TW`/`zh-CN`/`ja`), enforced by `locales/parity.test.ts`. Branch `feat/slide-manual-editing`. The full suite + real-API `:8000` gate run **once at the end** (Tasks 9–10), not per task.

## Acceptance criteria (verified at the real-API gate, Task 10)

- **A1 — Frame edit round-trips verbatim.** Toggle edit-current-frame on a content page, change the frame LaTeX, Save → the recompiled deck shows exactly that edit, page count/other frames unchanged, and the view stays on the edited page (no snap to page 1). The deck's other slides are byte-identical.
- **A2 — Whole-deck edit round-trips.** Edit-all-deck, change the title/preamble + a frame, Save → recompiles, all changes present, version snapshot created, grounding re-resolved.
- **A3 — Compile failure is non-destructive.** Introduce a deliberate LaTeX error (`\begin{frame}` with no `\end{frame}`), Save → the editor shows the `pdflatex` error log, the on-screen PDF is still the last-good deck (not blank), and a retry with valid LaTeX succeeds.
- **A4 — Sources strip resolves to the Canvas.** On a grounded content page the strip shows `paper §section` chips; clicking one opens the Citation Canvas highlighting a real chunk of that section. An unsourced page shows the muted chip; a title page shows the empty state.
- **A5 — Grounding survives a manual edit.** After a frame Save, the edited slide's `source_sections_json` is re-resolved from its `% cite:` marker (present → chips update; removed marker → empty, non-crashing).

pytest/Vitest prove mechanism; A1–A5 are proven live.

### Task 3: `_manual_recompile_and_persist` shared helper + `PUT /deck/slides/{page}/tex`

**Files:** Modify `api/decks.py`; extend the test file.

The helper takes the **candidate full deck tex** and persists it iff it compiles, keeping the last-good deck on failure. Mechanics (mirrors `_recompile_and_emit` but mechanical/no-LLM and non-destructive-on-fail):

- [ ] **Step 1 — failing tests** (stub `compile_mod.compile_with_revise` so no TeX Live needed):
  - success: `PUT /deck/slides/{page}/tex {frame_tex}` for a page → returns `{ok:true, page_count, status:"ok"}`; `deck_slides` rebuilt (the edited frame's row updated); a new `version_*.json` snapshot exists; `decks.current_version_id` bumped.
  - **compile failure keeps last good:** stub returns `CompileResult(ok=False, …, log="! Undefined control sequence")` → endpoint returns `{ok:false, status:"error", log:<tail>}`, and `deck.tex`/`deck.pdf` on disk are **unchanged** (assert bytes equal the pre-call deck).
  - page→slide resolution: editing a **continuation page** (`page_start < page ≤ page_end`) targets the owning slide's frame.
  - 404 when no deck / no slide covers the page.
- [ ] **Step 2 — run** → FAIL.
- [ ] **Step 3 — implement:**
  - `_manual_recompile_and_persist(conn, *, session_id, candidate_tex, description, preserved_notes)`:
    1. `slides_dir = _slides_workdir(session_id)`; read current `deck.tex` bytes (last-good).
    2. `compile_with_revise(tex=candidate_tex, workdir=slides_dir, tex_name="deck_candidate.tex", revise=_identity_revise, max_retries=1)` — compiles to `deck_candidate.pdf` in the SAME workdir (assets resolve) **without touching `deck.tex`/`deck.pdf`**.
    3. **fail:** clean up `deck_candidate.{tex,pdf,aux,log}`; return `(ok=False, log_tail)`. `deck.tex`/`deck.pdf` are untouched (we never wrote them).
    4. **ok:** write `result.tex` → `deck.tex`; move/copy `deck_candidate.pdf` → `deck.pdf`; `VersionHistory(slides_dir).save_version(result.tex, description, preserved_notes)`; `upsert_deck(... page_count=result.page_count, status="ok", current_version_id=new_version_id)`; `UPDATE runs… deck_version_id` skipped (no run for a manual edit — pass `run_id=None`); `replace_deck_slides(conn, deck_id, await with_grounding(build_deck_slides(result.tex, result.page_count), result.tex, conn))`; reapply `preserved_notes` by `slide_index` via `update_slide_note`; `rebuild_speaker_notes_json`. Return `(ok=True, page_count)`.
  - `PUT /deck/slides/{page}/tex`: resolve slide covering `page` (the `edit_deck_note` lookup); capture current per-slide notes (`{idx: note_text}` for all rows, so untouched frames keep notes; the edited frame's note is **preserved too** — a manual frame edit doesn't invalidate its note unless the user cleared it); `candidate = splice_frame(deck_tex, target.frame_tex, body.frame_tex)`; call the helper; on failure return `{ok:false, status:"error", log}` (HTTP 200 — a compile error is a normal outcome the editor renders, not a 4xx).
- [ ] **Step 4** — ruff + mypy; commit `feat(api): PUT /deck/slides/{page}/tex — manual frame edit + recompile`.
