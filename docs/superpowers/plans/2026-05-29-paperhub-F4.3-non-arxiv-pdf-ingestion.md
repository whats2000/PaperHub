# PaperHub F4.3 ŌĆö Non-arXiv PDF Ingestion via Unpaywall Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make papers that exist on publishers' sites but not on arXiv ingestible. Today, when Semantic Scholar resolves a paper with `arxiv_id=null` AND `openAccessPdf.url=null`, the dispatcher raises `NoIngestibleSourceError` ŌĆö even when a free PDF exists at the publisher (Nature, Science, conference proceedings, ŌĆ”). The chat layer surfaces the card with `auto_added=false, error="no_ingestible_source"` and the user sees just polite prose ("µłæÕĘ▓ńČōńé║µé©µēŠÕł░") with no usable result.

**Motivation (real-world trigger):** Live run 284, session 163, user message `"Can you find the paper of alphagenone?"`. The Resolver correctly identified Google DeepMind's *AlphaGenome: advancing regulatory variant effect prediction with a unified DNA sequence model* (Nature, 2025), but the SS metadata carried `arxiv_id=null` and empty `open_access_pdf_url`. Three SS paperIds for the same paper, all the same gap. AlphaGenome is publicly free on nature.com ŌĆö the URL just isn't in SS's index. Probing SS directly via the project's `search_papers` client confirmed: `pdf=''` for all three variants.

**Architecture:** Add **Unpaywall** as a secondary "find me a free PDF for this DOI" service the `ss:` branch falls back to when SS has no open-access URL. Unpaywall (`https://api.unpaywall.org/v2/{doi}?email=<contact>`) is free, no API key, and specifically built for "what's the best open version of this DOI" ŌĆö it covers Nature/Science OA articles, conference proceedings, preprint mirrors, and institutional OA repositories. The new dispatcher branch flows:

```
ss:<paperId> ŌåÆ fetch SS meta
  if meta.arxiv_id           ŌåÆ existing _attach_arxiv   Ō£ō
  elif meta.open_access_pdf_url ŌåÆ existing _attach_pdf  Ō£ō
  elif meta.doi AND settings.unpaywall_email:           Ō¤Ą NEW (F4.3)
      oa_url = await find_oa_pdf_by_doi(meta.doi, email=...)
      if oa_url:
          synthesized = dataclasses.replace(meta, open_access_pdf_url=oa_url)
          return await _attach_pdf(synthesized, ...)
  raise NoIngestibleSourceError      ŌåÉ still the residual; UX hint covered in ┬¦UX
```

Key properties:

- **Graceful degradation.** When `PAPERHUB_UNPAYWALL_EMAIL` is unset, the new branch is skipped entirely ŌĆö behaviour reverts to F4.2. Same "optional external" posture as `pandoc` / `open-websearch` / `marker`.
- **No new core dependencies.** `httpx` is already in the lock; `respx` is already a test dep. Unpaywall errors (404, 429, 5xx, transport) all soft-fail to `None` ŌĆö never break `/papers` POST or chat auto-attach when the user wasn't relying on Unpaywall in the first place.
- **No change to ingest pipeline.** `_attach_pdf` already accepts a `SemanticScholarMetadata` with an `open_access_pdf_url` and pipes the URL into `ingest_pdf_from_url`, which fetches via httpx, content-type-checks PDF, sha256-keys, and runs the standard PaperPipeline. We just give it a URL from a different source.
- **DOI is the join key.** SS already returns DOIs in `externalIds.DOI`; we just don't extract them. Adding `doi: str | None` to `SemanticScholarHit` and `SemanticScholarMetadata` (and threading through `_coerce_hit` / `_coerce_metadata`) is mechanical.

**Tech Stack:** Python 3.11, httpx + respx, `aiosqlite`. New env var: `PAPERHUB_UNPAYWALL_EMAIL`. No new core deps. No DB schema change. No LLM-visible contract change.

## Background / current state (read before starting)

- **The `ss:` dispatch fallback chain is the load-bearing path.** [`backend/src/paperhub/agents/research_tools.py:503-529`](../../../backend/src/paperhub/agents/research_tools.py#L503-L529): the dispatcher already tries `arxiv_id` first (richer LaTeX-source ingest path), falls back to `open_access_pdf_url`, raises `NoIngestibleSourceError` when neither is present. F4.3 inserts a third try (Unpaywall) between the second and the raise. **Do not reorder.** The order matters: arXiv source has highest fidelity (real figures + captions + equations as LaTeX); SS-provided OA URL is second-best (publisher PDF, but still authoritative); Unpaywall-resolved OA URL is third-best (could be any OA mirror ŌĆö the publisher copy is usually preferred when both exist, and Unpaywall returns the best one via `best_oa_location`).

- **SS metadata extraction lives in two coerce functions.** [`backend/src/paperhub/pipelines/semantic_scholar.py:180-203`](../../../backend/src/paperhub/pipelines/semantic_scholar.py#L180-L203): `_coerce_hit` builds a `SemanticScholarHit` from a raw SS dict; `_coerce_metadata` calls `_coerce_hit` and re-wraps as `SemanticScholarMetadata`. DOI extraction needs adding to `_coerce_hit` so both the `/search` (returns hits) and `/paper/{id}` (returns metadata) paths get DOIs.

- **`SemanticScholarHit` and `SemanticScholarMetadata` are intentionally similar.** The split exists so call sites document intent (single-paper lookup vs. search-result list); both should grow `doi: str | None` together.

- **`_attach_pdf` is at `backend/src/paperhub/agents/research_tools.py:434-455`.** It takes a `SemanticScholarMetadata` and pipes `meta.open_access_pdf_url` into `pipeline.ingest_pdf_from_url`. It asserts `open_access_pdf_url is not None` ŌĆö so the Unpaywall branch must construct a meta with the URL injected BEFORE calling `_attach_pdf`. `dataclasses.replace(meta, open_access_pdf_url=oa_url)` is the idiomatic move (the dataclass is `frozen=True`).

- **Settings already follow the env-with-default pattern.** [`backend/src/paperhub/config.py:7-87`](../../../backend/src/paperhub/config.py#L7-L87). Add `unpaywall_email: str | None` to the dataclass and `unpaywall_email=os.environ.get("PAPERHUB_UNPAYWALL_EMAIL") or None` to `load_settings()`. `None` means "fallback disabled"; never raise on missing.

- **Dispatcher callers already hold a `Settings`.** Two sites call `add_paper_to_session_dispatch`:
  - `backend/src/paperhub/api/papers.py:157` (POST /papers) ŌĆö has `settings = load_settings()` two lines above.
  - `backend/src/paperhub/api/chat.py:253` (chat auto-attach) ŌĆö already passes `settings` around the request scope.

  Both can pass `unpaywall_email=settings.unpaywall_email` without restructuring.

- **The Unpaywall API contract.** `GET https://api.unpaywall.org/v2/{doi}?email=<contact>`. Returns 200 with JSON `{is_oa, best_oa_location, ...}` on success; 404 when the DOI is unknown to Unpaywall; 422 when `email` is missing; 429 under sustained load. `best_oa_location` is the JSON object `{url, url_for_pdf, ...}`; the URL we want is `url_for_pdf` (a direct PDF) when present, else `url` (a landing page that the pipeline's content-type check will accept or reject ŌĆö Marker handles both cases).

- **Real-`:8000` is the verification gate.** The unit tests below (respx-mocked Unpaywall + envelope-shaped SS stub) prove wiring + control flow; the live verification step proves the end-to-end flow against real SS + real Unpaywall + real Marker actually delivers the paper to the user. Both are required to mark this complete ŌĆö pytest green is necessary-but-insufficient (see CLAUDE.md "Backend quality gates" ┬¦pytest-measures-syntax).

### Task A2: Implement the client

**Files:**
- Create: `backend/src/paperhub/pipelines/unpaywall.py`

- [ ] **Step 1:** Module skeleton:

  ```python
  """Unpaywall OA-PDF lookup (F4.3).

  Secondary "find me a free PDF for this DOI" service used by the
  ``ss:`` dispatch when SS itself didn't carry an ``openAccessPdf.url``.
  Free, no API key ŌĆö requires only an email for abuse-control logging.
  Soft-fails on every error path so an Unpaywall outage never breaks
  the dispatcher's outer try/raise contract.
  """
  from __future__ import annotations

  import logging
  from typing import Any

  import httpx

  UNPAYWALL_BASE = "https://api.unpaywall.org/v2"
  _TIMEOUT_S = 8.0

  _LOG = logging.getLogger(__name__)


  async def find_oa_pdf_by_doi(doi: str, *, email: str) -> str | None:
      """Return the best open-access PDF URL Unpaywall knows for `doi`, or
      None if no OA version is available / Unpaywall is down / the DOI is
      unknown. Never raises ŌĆö the dispatcher uses None to mean
      'no URL; fall through to NoIngestibleSourceError'."""
      url = f"{UNPAYWALL_BASE}/{doi}"
      params = {"email": email}
      try:
          async with httpx.AsyncClient(timeout=_TIMEOUT_S) as client:
              resp = await client.get(url, params=params)
      except httpx.HTTPError as exc:
          _LOG.info("unpaywall transport error doi=%s err=%s", doi, exc)
          return None
      if resp.status_code != 200:
          _LOG.info(
              "unpaywall non-200 doi=%s status=%d", doi, resp.status_code,
          )
          return None
      try:
          data: dict[str, Any] = resp.json()
      except ValueError:
          return None
      if not data.get("is_oa"):
          return None
      loc = data.get("best_oa_location") or {}
      if not isinstance(loc, dict):
          return None
      return loc.get("url_for_pdf") or loc.get("url") or None
  ```

## Known follow-ups (out of F4.3 scope)

1. **Metadata-only card with "Upload PDF" CTA.** When BOTH SS and Unpaywall have nothing (gated journal, no preprint, no OA mirror), the user still sees the polite-prose-no-card outcome. A future F4.4 increment would emit the resolved metadata as a card with a frontend "Upload PDF" affordance reusing the existing `/papers/upload` endpoint ŌĆö covering the residual long tail. Out of scope here because it's a frontend change + a new card-state signal, not a backend ingest improvement.

2. **DOI-based publisher direct fetch.** Some publishers expose a stable `https://<publisher>.com/articles/<DOI>.pdf` pattern. Building per-publisher resolvers is brittle and out of scope; Unpaywall already aggregates this knowledge across publishers, and we accept the long-tail miss rate.

3. **Cache Unpaywall lookups.** A given DOI's OA URL doesn't change minute-to-minute. A small SQLite cache (one row per DOI, TTL ~24h) would save Unpaywall round-trips when users re-ask about the same paper across sessions. Defer until we have a measured hot-spot ŌĆö the SS rate-limit budget is the tighter constraint today.
