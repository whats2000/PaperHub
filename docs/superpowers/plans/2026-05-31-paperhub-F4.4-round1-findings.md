## The architectural win

Round-0's failure analysis localized two structural bombs:

1. **`sl_narrate` figure-inventory bomb.** Round-0's narrate prompt received the full per-paper figure inventory verbatim. For figure-heavy papers (FASTer, 13 figures) and multi-paper (3 papers, ~25-30 figures combined), the prompt drags past Gemini's per-call latency threshold and disconnects. 2 retries on FASTer both failed identically.
2. **`sl_assemble` truncation.** Multi-ZH's narrate survived (51s), drafts ran, coherence ran, but assemble produced a 3-frame deck.tex with the last frame unclosed; pdflatex failed; revise loop couldn't recover from structural truncation.

Both modes are fully solved by Round 1's agentic-brief + bounded-fallback hybrid topology:

- `sl_paper_brief` extracts only top 3-5 figures per paper with one-line interpretations into a dense `PaperTalkBrief`. Bounded per-paper context (~5-10k tokens each). The raw inventory never reaches the planner.
- `sl_plan_deck` consumes N briefs (~25k for 3 papers — fits one call easily). Narrate-disconnect mode goes away.
- `sl_render_slide` per-slide fan-out emits ONE well-formed `\begin{frame}...\end{frame}` per call. Assemble becomes a simple concat of validated frames, removing truncation risk.

The Round-1 traces confirm: FASTer's 3 paper_briefs ran in 28-40s each, plan_deck in 36s, 13 render_slides total in ~3 min; multi-EN got through 3 briefs + plan + 15 render_slides before the planner-validator caught the prompt ambiguity. Compare to Round-0 FASTer dying at 60s in `report:narrate` with zero deck content.

## Observable changes between rounds (developer notes, not scores)

Three contract-level capabilities the new chain wires up that the OLD chain didn't have. These are observable in the emitted tex; whether they translate to a better deck for the audience is a human call.

1. **Equation + notation explanation paired in math slides.** The new T3 `math_stack` template includes a `\textbf{Header:}` + `\[...\]` + symbol-listing block; the planner attaches a `notation_explanation` field on every `key_equation` from the brief. The OLD chain dropped raw equations with no symbol guide.
2. **`\input{ADDITIONAL.tex}` equivalent.** T4's `% BEGIN paperhub:paper_newcommands` block is in every deck preamble, populated from each paper's brief. OLD chain had no analog; math using a paper's `\newcommand`s could break silently.
3. **Theme is yaml-driven.** T8's `slide_style_profiles.yaml :: default` ships Berlin/dolphin/professionalfonts/14pt/16:9 + custom footline + accent colors. Operators edit the yaml to change defaults; F4.2's `edit_preamble` chat path handles per-session overrides. Templates are REFERENCE LAYOUTS, not enforcement (T10).

Plus the multi-paper scenarios gain:
- **Skeleton order** (title → references → bottlenecks_table → per-paper concept+math → proposed_direction → plan_numbered → takeaway_closer) matches the gold's exact pattern sequence.
- **Bottlenecks-overview connective tissue** — 3-row booktabs mapping each paper to one axis of the shared problem.
- **References slide** — 3-column `[s]`-stretch minipage with per-paper tag + italic title + venue badge.
- **Takeaway_closer** — `\rule` framing + take-away sentence + Open Question block + Thank you.

## What broke in a tractable way

**Multi-EN failed at `plan_validation_failed`** — the planner emitted a `motivation_figure` PlannedSlide at `slide_index=2` with `paper_id=null`. T2's validator (added in T2's review-fix to enforce cross-paper attribution rules) correctly raised. The bug:

- `motivation_figure` is currently classified as a per-paper pattern (it shows ONE paper's motivation figure).
- For multi-paper decks, the planner correctly identifies "we need a motivation slide" but didn't pick WHICH paper's motivation figure to use.
- Multi-ZH happened to skip the motivation_figure (went straight to bottlenecks_table) and succeeded.

**Round-2 fix is small:** either (a) tighten the `slides_plan_deck_v1.yaml` prompt for multi-paper case to require explicit paper_id attribution on `motivation_figure`, OR (b) introduce a `cross_paper_motivation` pattern variant that allows `paper_id=null` and renders a shared motivation slide. Either is a few hours of work.

## Closeout lessons + harness improvement

The Round-1 closeout re-run surfaced TWO harness-design issues worth recording:

1. **`decks.theme` stale-row persistence on regenerate.** Decks first created in earlier rounds (28, 29) retained their original `theme` column value when re-generated under T8 — even though the actual emitted tex IS the new yaml-driven gold preamble. The DB column is purely informational (no lookup); the user-visible deck quality is correct. T8's implementer noted this is by-design (decks.theme stores the canonical name of the resolved profile); the cosmetic mismatch is a stale row-update issue in `upsert_deck`. **Round-2 fix: `upsert_deck` should refresh `theme` on every regenerate.**

2. **The deck_command classifier routes "prepare slides..." to NOTES when an existing deck is present.** When the harness re-ran multi-EN against the persistent session 175 (which already had a multi-ZH deck from an earlier run), the deck_command classifier saw "prepare a 12-minute conference talk" + existing-deck and routed to `report:note_author` (speaker notes generation), NOT to regenerate. Multi-ZH (different prompt language) went to regenerate. The harness mitigation: **fresh session per scenario** to guarantee no-deck-state on each run. The closeout re-runs use this approach (re-seeded session 176). **Round-2 fix: tighten deck_command classifier so "prepare a deck"-style prompts route to regenerate unambiguously, OR add a `--force-regenerate` flag to the harness prompts.**

## Known follow-ups for Round 2 (must-fix)

1. **Multi-paper `motivation_figure` attribution** (intermittent). The planner sometimes emits `motivation_figure` on multi-paper with `paper_id=null`, triggering T2's `_validate_attributions` raise. Round-1 first-pass multi-EN hit it; closeout multi-EN didn't. Either prompt-tighten or add a `cross_paper_motivation` pattern variant.
2. **`decks.theme` stale-row persistence** (see closeout lessons above).
3. **Deck_command classifier ambiguity on "prepare slides" with existing deck** (see closeout lessons above).
4. **Deck-version history (currently MISSING).** Each generate-slides turn OVERWRITES the session's deck row (one deck per session, upserted on regenerate). Older `deck.tex` files are lost. No way to compare across iterations or roll back. Round 2 design:
   - Allow multiple `decks` rows per session. `decks.session_id` becomes a regular index, not a unique key. Add `decks.is_current: bool` (or sort by `created_at DESC` to derive the current pointer).
   - Each generate-slides chat turn emits a persistent slide card in the chat message history (same surface F4.2 / F4.3 already use for other artifacts). The card carries its `deck_id`.
   - **Inactive versions get a "Switch to this version" button** on the slide card.
   - **Switch flow (deterministic, no LLM):** click → backend rewrites `chat_session/<sid>/slides/deck.tex` to the selected version's tex (also copies its `deck.pdf` if present, else recompiles) → flips `decks.is_current=true` on the target row, `false` on the previously-active row → the Slides panel refreshes to render the activated deck.
   - Storage: per-deck workspace dir `chat_session/<sid>/decks/<deck_id>/deck.{tex,pdf}` instead of the shared `chat_session/<sid>/slides/`. The shared dir becomes a symlink / synced copy of the currently-active deck for backward-compat with code that reads the legacy path.
   - The Slides panel gets a dropdown / breadcrumb showing all versions for the session (timestamped).
   - F4 NOTES + F4.2 EDIT flows operate on the currently-active deck only; switching versions doesn't lose the inactive versions' notes (each deck row keeps its own `speaker_notes_json`).
5. **`SlideStyleProfile` active-memory persistence + `set_style` chat-command action.** T8 shipped the yaml registry + dataclass + env-var + per-session override via F4.2's existing `edit_preamble`. Round 2 should add (a) an active-memory carrier so "always use the casual_talk profile" gets remembered across sessions, and (b) a new deck-command action `set_style` so a chat turn like "switch to the casual_talk profile" routes directly to a profile swap (not through `edit_preamble` LLM rewrite). New profiles (`casual_talk`, `defense_long`, `lightning_5min`, …) drop into the yaml without code changes — that's the T8 contract.
6. **Stale OLD chain prompts + DEPRECATED functions in `report_graph.py`.** T5 left them with `# DEPRECATED` markers. After Round 2 confirms stability, cleanup commit removes them + the 4 unused prompt YAMLs.
7. **ChkTeX-driven revise loop.** Round-1 closeout still saw occasional Vertex disconnects on `report:compile`'s LLM-driven revise calls. Restoring the OLD project's ChkTeX-feedback path would catch escape-char / ampersand / undefined-macro errors earlier with deterministic signal instead of LLM-only.
8. **Figure-aspect-aware pattern selection.** The current planner picks `concept_2col` (figure-left 0.55 + text-right 0.45) regardless of the actual figure's aspect ratio. Wide-format figures get cramped; portrait figures fit the 2-column. Fix: thread the figure's dimensions through `PaperTalkBrief.key_figures.KeyFigure` (read from F2's `PaperAsset`), and have the planner choose `concept_horizontal` (figure on top or bottom, text in the other half) for wide figs vs `concept_2col` for portrait figs. New `pattern_kind` values: `concept_figure_on_top`, `concept_figure_on_bottom`. Each goes into `slide_pattern_templates.yaml` as a separate reference layout.
9. **Harness scoring redesign.** The Round-1 scorecards were pattern-matched against the example deck's surface form (dimension-by-dimension fit-with-gold), which is self-fulfilling for a chain whose templates encode the gold's exact layout. Future rounds: the harness dumps OBJECTIVE machine-checks (compile, figure resolution, frame count, language) and opens PDFs for HUMAN review. No 0/1 dimension scoring against the example. **Already implemented in this commit.**
