# PaperHub F4.2 — User Slide-Style Customization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give users a conference-grade, metadata-driven title page by default, and let them customize their deck — the title page's content + layout, and deck-wide style (theme, colors, fonts, header/footer, layout) — through chat.

**Architecture:** Builds on F4 (SRS v2.21). GENERATE now fills the Beamer preamble with paper metadata (`\title`/`\author`/`\date`/`\institute`) and renders the title page as a **real, editable `\titlepage` frame** instead of bare `\maketitle`. Two new deck-command actions add focused edit sub-flows: **`edit_title`** (the title page's metadata + that frame's layout) and **`edit_preamble`** (deck-wide style — theme, colors, fonts, header/footer, packages). Both reuse the existing `get_preamble`/`replace_preamble` "page-1 block" helpers and the existing Overfull-aware recompile loop; neither touches content `deck_slides` text, and both preserve speaker notes by `slide_index`.

**Tech Stack:** Python 3.11, LangGraph subgraph (`agents/report_graph.py`), LiteLLM adapter + YAML prompt slots, Beamer/pdflatex(+xelatex) compile pipeline, aiosqlite. Tests: pytest (mock adapter + `fake_tracer`/`migrated_db` fixtures).

## Background / current state (read before starting)

- **Title page today:** `assemble_deck` (`backend/src/paperhub/pipelines/slide_pipeline/assemble.py`) emits `\title{<talk title>}` then `\begin{document}` + `\maketitle`. `\author`/`\date` are never set, so `\maketitle` renders a bare title. Metadata (`paper_content.authors_json`, `year`, `arxiv_id`) is **not** loaded into the slide pipeline.
- **`_enabled_papers`** (`agents/report_graph.py`) selects only `id, title, abstract, sections_json, source_dir_path`.
- **Page-1 / title-page editing is impossible today:** the title page has no `deck_slides` row (content rows start at page 2), and `_select_rows` scope=`current` on page 1 falls back to the first content slide. There is no path to edit `\title`/theme.
- **Existing reusable helpers** (`pipelines/slide_pipeline/beamer_helpers.py`):
  - `get_preamble(beamer_code) -> str | None` — returns the "page-1 source block": the preamble, **plus** the first frame when it is a real `\titlepage` frame (for the `\maketitle` case, preamble only).
  - `replace_preamble(beamer_code, new_block) -> str | None` — swaps that page-1 block, leaving every later frame untouched.
  - `extract_frames_from_beamer`, `replace_frame_in_beamer`.
- **Indexing:** `build_deck_slides` (`pipelines/slide_pipeline/deck_slides_map.py`) drops the synthetic `\maketitle` tuple and absorbs one extra leading page-group; `_real_frame_number` (`agents/report_graph.py`) skips the synthetic `\maketitle` so content `slide_index` 0 → the first real content frame. **Introducing a real `\titlepage` frame must be skipped the same way** so content indexing (and thus notes/edit_frame mapping) is unchanged.
- **`paper2slides-plus` reference** (`reference/paper2slides-plus/prompts/config.yaml`): line 49 (title-page metadata: paper title + first author surname + "et al." + arXiv release date, placed in the **preamble**), and `interactive_edit_preamble` (lines 210–246) — the model rewrites the preamble (`\documentclass`/`\usepackage`/`\usetheme`/`\title`/`\author`/`\date`/style) + title-page frame, preserving structure and valid themes. Our two edit prompts are modeled on these.
- **Recompile/emit template:** `_edit_slides` in `agents/report_graph.py` (the block after frame edits — `verify_and_fix_graphics` → `compile_with_revise` → `VersionHistory.save_version` → `upsert_deck` → `replace_deck_slides` → restore notes by `slide_index` → `rebuild_speaker_notes_json` → `_emit_deck`). The two new nodes reuse this block verbatim except for how `new_tex` is produced.

```python
"""Derive Beamer title-page metadata (title / author / date) from the
contributing papers (PaperHub F4.2). Pure + unit-tested.

Single paper  → the paper's own title, its authors (surnames, "et al." past 3),
                and "arXiv:<id> · <year>" (or just <year>).
Multiple      → the LLM talk title, with each paper's lead-author surname listed.
All fields are LaTeX-escaped for safe interpolation into \\title/\\author/\\date.
"""
from __future__ import annotations

from dataclasses import dataclass

_LATEX_SPECIALS = {
    "\\": r"\textbackslash{}", "&": r"\&", "%": r"\%", "$": r"\$",
    "#": r"\#", "_": r"\_", "{": r"\{", "}": r"\}", "~": r"\textasciitilde{}",
    "^": r"\textasciicircum{}",
}


def latex_escape(text: str) -> str:
    """Escape LaTeX special characters in plain text (titles, names)."""
    out: list[str] = []
    for ch in text:
        out.append(_LATEX_SPECIALS.get(ch, ch))
    return "".join(out)


def _surname(full_name: str) -> str:
    """Last whitespace-delimited token of a 'First Last' name."""
    parts = full_name.strip().split()
    return parts[-1] if parts else full_name.strip()


def _authors_str(authors: list[str], max_names: int = 3) -> str:
    surnames = [_surname(a) for a in authors if a.strip()]
    if not surnames:
        return ""
    if len(surnames) > max_names:
        return ", ".join(surnames[:max_names]) + ", et al."
    return ", ".join(surnames)


def _date_str(year: int | None, arxiv_id: str | None) -> str:
    bits: list[str] = []
    if arxiv_id:
        bits.append(f"arXiv:{arxiv_id}")
    if year:
        bits.append(str(year))
    return " · ".join(bits)


@dataclass(frozen=True)
class TitleMetadata:
    title: str       # LaTeX-escaped
    author: str      # LaTeX-escaped
    date: str        # LaTeX-escaped (already safe; arXiv ids/years have no specials)


def build_title_metadata(
    papers: list[dict[str, object]], *, talk_title: str
) -> TitleMetadata:
    if not papers:
        return TitleMetadata(title=latex_escape(talk_title), author="", date="")
    if len(papers) == 1:
        p = papers[0]
        title = str(p.get("title") or talk_title)
        author = _authors_str([str(a) for a in (p.get("authors") or [])])
        date = _date_str(p.get("year") if isinstance(p.get("year"), int) else None,
                         str(p["arxiv_id"]) if p.get("arxiv_id") else None)
        return TitleMetadata(
            title=latex_escape(title),
            author=latex_escape(author),
            date=latex_escape(date),
        )
    # Multi-paper: talk title + lead-author surname per paper.
    leads: list[str] = []
    for p in papers:
        a = [str(x) for x in (p.get("authors") or [])]
        if a:
            leads.append(f"{_surname(a[0])} et al.")
    return TitleMetadata(
        title=latex_escape(talk_title),
        author=latex_escape(" · ".join(leads)),
        date="",
    )
```

> Note: `build_title_metadata` takes `talk_title` as a keyword arg. In the multi-author test the surname of "A_B Smith" is "Smith" (no underscore), matching `test_escaping_applied_to_title_and_author`.

def is_title_frame(frame_content: str) -> bool:
    """True for the deck's title page: the synthetic ``\\maketitle`` marker, or a
    real frame whose body is essentially ``\\titlepage`` (no content bullets). It
    is excluded from content slide indexing and edited via the title/preamble
    sub-flows, never as a content frame."""
    s = frame_content.strip()
    if s == r"\maketitle":
        return True
    return "\\titlepage" in s and "\\itemize" not in s and "\\includegraphics" not in s
```

@dataclass
class AssembleInput:
    title: str
    theme: str
    additional_tex_macros: list[str]
    cache_source_dirs: list[str]
    frames: list[str]
    author: str = ""
    date: str = ""
    subtitle: str = ""


def assemble_deck(inp: AssembleInput) -> str:
    preamble: list[str] = [
        "\\documentclass{beamer}",
        f"\\usetheme{{{inp.theme}}}",
        "\\usepackage{graphicx}",
        "\\usepackage{booktabs}",
        "\\usepackage{amsmath,amssymb}",
        build_graphicspath(inp.cache_source_dirs),
        build_additional_block(inp.additional_tex_macros),
        f"\\title{{{inp.title}}}",
    ]
    if inp.subtitle:
        preamble.append(f"\\subtitle{{{inp.subtitle}}}")
    if inp.author:
        preamble.append(f"\\author{{{inp.author}}}")
    if inp.date:
        preamble.append(f"\\date{{{inp.date}}}")
    parts: list[str] = [
        *preamble,
        "\\begin{document}",
        # Real, editable title frame (not bare \maketitle) so its layout can be
        # customized via the edit_title sub-flow (F4.2).
        "\\begin{frame}[plain]\n\\titlepage\n\\end{frame}",
        *inp.frames,
        "\\end{document}",
    ]
    return "\n".join(p for p in parts if p) + "\n"
```

def _real_frame_number(full_tex: str, slide_index: int) -> int | None:
    """Map a 0-based content ``slide_index`` to the 1-based frame number
    ``replace_frame_in_beamer`` expects, skipping a leading title frame

async def _edit_page_block(
    *, adapter: LlmAdapter, tracer: Tracer, model: str, slot: str, tool: str,
    page_block: str, instruction: str, response_language: str,

    async with tracer.step(agent="report", tool=tool, model=model) as step:

1. **Change the slides layout, header/footer, etc.** → `edit_preamble` (`\setbeamertemplate{headline}/{footline}/{frametitle}`, outer theme, margins) — Tasks B4/B5/B6/B7, prompt enumerates header/footer.

## Known follow-ups (out of scope)

- **Per-slide style overrides** (one frame in a different color) remain `edit_slides`; deck-wide style is `edit_preamble`.
- **Theme allow-listing / validation:** the prompt asks for valid Beamer themes; an invalid theme surfaces as a compile error handled by the revise loop (no deterministic theme allow-list yet).
- **Title-page editability when the deck predates F4.2** (old decks compiled with bare `\maketitle`): `get_preamble` returns the preamble only, so `edit_title` edits metadata but there is no title *frame* to re-layout until the deck is regenerated. Acceptable; note in the SRS.
